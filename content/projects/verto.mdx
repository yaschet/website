## The Problem

Businesses need to translate documents. Not just words, but entire layouts: contracts, reports, product manuals. Tables, headers, formatting. The existing options were either too simple (Google Translate) or too expensive (human agencies).

The client wanted something in between: AI-powered translation that actually preserves how a document looks.

---

## What I Built

<Image
  src="/images/verto/dashboard-page.png"
  alt="Verto dashboard showing document queue and translation status"
/>

Verto is a full SaaS platform. Users upload a document, pick a language, and get back a translated version that looks like the original. Tables stay tables. Headers stay headers. The complexity happens behind the scenes.

### The Stack

| Layer           | Tech              | Why                                    |
| --------------- | ----------------- | -------------------------------------- |
| Frontend        | Next.js 16        | Server components, fast, modern        |
| Auth            | Supabase          | Email, magic links, MFA out of the box |
| Database        | PostgreSQL + RLS  | Multi-tenant data isolation            |
| AI              | OpenAI GPT-4o     | Translation and document analysis      |
| Background Jobs | Trigger.dev       | Long-running document processing       |
| Payments        | Stripe            | Credits and subscriptions              |

## Hard Problems I Solved

### Large Documents

**The issue**: Documents over 50 pages crashed the system. Memory limits, timeouts, the usual.

**The fix**: Chunked processing. Each section processes independently. If one fails, it retries. The rest keeps going.

### Data Isolation

**The issue**: Multiple companies on one platform. Their data can never mix.

**The fix**: PostgreSQL Row-Level Security. Every query is automatically scoped to the user's organization. No code changes needed. It just works.

### Translation Quality

<Compare
  before="/images/verto/landing-page-blur.png"
  after="/images/verto/landing-page.png"
  caption="Before and after: layout preservation in action"

**The issue**: AI translations lose context between sections. Terms get inconsistent.

**The fix**: Custom prompts with sliding context windows. A glossary system for industry terms. Quality went from "okay" to "actually usable."

## What I Learned

1. **Auth is not optional**: MFA, session management, password policies. Do it right from day one.

2. **Background jobs fail**: Build for it. Durable execution. Automatic retries. Idempotent operations.

3. **RLS is powerful**: But you have to test it. A lot.

4. **Prompts are products**: The difference between good and great AI output is in the prompt details.

5. **Credits beat subscriptions**: For usage-based services, people prefer paying for what they use.
